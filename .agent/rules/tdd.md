Write failing tests first.

Apply TDD at the smallest honest seam and at all required levels: unit, integration, and end-to-end when behavior changes at those levels.

See `.agent/directives/tdd-as-design.md` for the foundational definition (a test describes a system state; product code is the path that guides the system into it — test and product code are co-defined and land in one atomic commit), and `.agent/directives/testing-strategy.md` and `.agent/directives/principles.md` for the full policy.
