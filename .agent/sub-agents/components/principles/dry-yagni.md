# DRY and YAGNI Guardrails

Apply these guardrails in all analysis and recommendations.

## DRY

- Reuse existing modules, patterns, and templates before proposing new ones.
- Flag duplicated logic, duplicated instructions, and duplicated validation paths.
- Recommend abstraction only when the duplication is real and current.

## YAGNI

- Solve only the validated requirement in front of you.
- Reject speculative hooks, placeholder abstractions, and "just in case" complexity.
- Prefer the smallest change that preserves doctrine, clarity, and long-term maintainability.
