# Plan

Create or revise a decision-complete plan for the next atomic slice of work.

## Steps

1. Read `AGENT.md`, `principles.md`, `testing-strategy.md`, `requirements.md`, and `DEFINITION_OF_DONE.md`.
2. Identify the real active or paused workstream entrypoint before drafting.
3. State:
   - the user impact to optimize for
   - scope and out-of-scope
   - assumptions to validate
   - measurable success criteria
   - TDD order
   - documentation outputs
   - execution trigger or completion criteria
4. Keep architecture truth in durable docs, not only in plans.
