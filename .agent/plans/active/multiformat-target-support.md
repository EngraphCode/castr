in the case of the input being openapi 3.1/2 and the output being a collection of Zod, TypeScript and Json Schema for supporting MCP tools, there is a particular challenge. Because Zod doesn't natively support example/s if we transform openapi -> zod -> json schema we risk losing the examples . We could absolutely store examples in Zod metadata, that is idiomatic <https://zod.dev/json-schema?id=metadata> , we could also configure an output target that consists of Zod AND TypeScript AND Json Schema, and route the data to the appropriate output format. What should we do about this, and where should we record that research?
